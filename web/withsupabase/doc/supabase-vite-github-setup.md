# Supabase + Vite + GitHub Pages Setup Guide

This guide shows how to create a Vite React app, connect it to Supabase, and deploy it to GitHub Pages.

## 1. Create a Vite React app

Open a terminal and run:

```bash
npm create vite@latest my-app -- --template react-ts
cd my-app
npm install
```

Start the app locally:

```bash
npm run dev
```

Open the local URL shown in the terminal.

## 2. Install Supabase

Inside the app folder, install the Supabase client:

```bash
npm install @supabase/supabase-js
```

## 3. Create a Supabase project

Go to Supabase:

- https://supabase.com

Create an account or sign in.

Then:

1. Click “New project”
2. Choose an organization
3. Enter a project name
4. Set a database password
5. Choose a region
6. Click “Create new project”

Wait for the project to finish creating.

## 4. Get your Supabase keys

In Supabase:

1. Go to Project Settings
2. Open “API”
3. Copy:
   - Project URL
   - anon/public key

Create a file named `.env` in your Vite app folder:

```env
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key
```

Important:
- Do not commit `.env` to GitHub
- Add `.env` to `.gitignore`

## 5. Connect Supabase in the app

Create a file like:

```ts
src/lib/supabase.ts
```

Add:

```ts
import { createClient } from '@supabase/supabase-js';

const supabaseUrl = import.meta.env.VITE_SUPABASE_URL || '';
const supabaseAnonKey = import.meta.env.VITE_SUPABASE_ANON_KEY || '';

export const supabase = createClient(supabaseUrl, supabaseAnonKey);
```

Now you can use Supabase anywhere in your app.

## 6. Enable authentication in Supabase

In Supabase:

1. Go to Authentication
2. Open “Settings”
3. Enable the auth methods you want
   - Email/password is the simplest
   - OAuth can be added later

If using email/password login, you usually do not need extra callback config for local development.

## 7. Create a simple table

In Supabase:

1. Go to Database
2. Open SQL Editor
3. Run a query like this:

```sql
create table if not exists products (
  id bigint generated by default as identity primary key,
  name text not null,
  price numeric not null,
  description text,
  image_url text
);
```

You can insert example data:

```sql
insert into products (name, price, description, image_url)
values
  ('Laptop', 999.99, 'Great for coding', 'https://example.com/laptop.jpg'),
  ('Phone', 699.99, 'Modern smartphone', 'https://example.com/phone.jpg');
```

## 8. Fetch data from Supabase

In your React component:

```ts
import { useEffect, useState } from 'react';
import { supabase } from './lib/supabase';

function App() {
  const [products, setProducts] = useState<any[]>([]);

  useEffect(() => {
    const loadProducts = async () => {
      const { data, error } = await supabase.from('products').select('*');
      if (error) {
        console.error(error);
      } else {
        setProducts(data || []);
      }
    };

    loadProducts();
  }, []);

  return (
    <div>
      <h1>Products</h1>
      {products.map((p) => (
        <div key={p.id}>
          <h3>{p.name}</h3>
          <p>{p.description}</p>
        </div>
      ))}
    </div>
  );
}

export default App;
```

## 9. Push the project to GitHub

Initialize Git:

```bash
git init
git add .
git commit -m "Initial commit"
```

Create a GitHub repository on GitHub.

Then connect it:

```bash
git branch -M main
git remote add origin https://github.com/your-username/your-repo.git
git push -u origin main
```

## 10. Deploy to GitHub Pages

### Step 1: Add GitHub Actions workflow

Create a folder:

```bash
mkdir -p .github/workflows
```

Create a file named:

```yaml
.github/workflows/deploy.yml
```

Use a workflow like this:

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

jobs:
  build:
    runs-on: ubuntu-latest
    environment:
      name: github-pages

    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 20

      - run: npm install

      - name: Build
        env:
          VITE_SUPABASE_URL: ${{ secrets.VITE_SUPABASE_URL }}
          VITE_SUPABASE_ANON_KEY: ${{ secrets.VITE_SUPABASE_ANON_KEY }}
        run: npm run build

      - uses: actions/configure-pages@v5

      - uses: actions/upload-pages-artifact@v3
        with:
          path: ./dist

      - uses: actions/deploy-pages@v4
```

### Step 2: Add GitHub secrets

In GitHub:

1. Go to Settings
2. Open Environments
3. Create or select `github-pages`
4. Add these secrets:
   - `VITE_SUPABASE_URL`
   - `VITE_SUPABASE_ANON_KEY`

### Step 3: Enable GitHub Pages

In GitHub:

1. Go to Settings
2. Open Pages
3. Set Source to “GitHub Actions”

Then push your changes to GitHub.

## 11. Common issues

### Problem: app shows blank page
Check:
- `.env` values are correct
- the Vite app was rebuilt
- the GitHub secrets were added

### Problem: Supabase auth fails
Check:
- the URL is correct
- the anon key is correct
- auth methods are enabled in Supabase

### Problem: GitHub Pages shows 404
Check:
- the workflow ran successfully
- the Pages source is set to GitHub Actions
- the app path is correct

## 12. Best practices

- Keep secrets in `.env` locally
- Never commit `.env`
- Use `.env.example` as a template
- Add `.gitignore` entries for `.env` and `dist`

Example `.gitignore`:

```gitignore
node_modules
dist
.env
```

## Summary

You now know how to:
- create a Vite app
- connect it to Supabase
- store secrets safely
- build a React app
- deploy it to GitHub Pages
